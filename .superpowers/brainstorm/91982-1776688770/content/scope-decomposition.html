5 features → 3 independent sub-projects

Each builds on the previous. Pick where to start.

1

UI Polish + Crop Lookup (no database needed)

Pest images, step-by-step form, animated result, mobile layout. Plus crop selector (maize/wheat/tomato → filter pests). Pure frontend + minor backend. Ships fast.

⏱ Effort: Small  |  Impact: High visibility
2

Persistence + Report History (adds database)

H2 embedded database (zero config). Save every submission. View history page — list of past reports, click to expand. Foundation for dashboard.

⏱ Effort: Medium  |  Impact: Makes it a real app
3

Dashboard + Weather Context (builds on phase 2)

Stats page: total reports, most common pest, avg delivery time. Weather: use Open-Meteo API (free, no key) to show season/rainfall and flag high-risk pests.

⏱ Effort: Medium  |  Impact: Impressive demo feature

Recommendation: Start with Phase 1 — delivers the biggest visual upgrade immediately, no database complexity. Then Phase 2 adds persistence. Then Phase 3 layers on top. Each phase works on its own.
